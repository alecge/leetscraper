<h4 id="approach-1-revert-half-of-the-number-accepted">Approach #1 Revert half of the number [Accepted]</h4>
Intuition</strong></p>
The first idea that comes to mind is to convert the number into string, and check if the string is a palindrome, but
this would require extra non-constant space for creating the string which is not allowed by the problem description.</p>
Second idea would be reverting the number itself, and then compare the number with original number, 
if they are the same, then the number is a palindrome. However, if the reversed number is larger than <span class="maths katex-rendered">i</mi>n</mi>t</mi>.</mi>M</mi>A</mi>X</mi></mtext></mrow><annotation encoding="application/x-tex">text{int.MAX}</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height: 0.68333em;"></span><span class="strut bottom" style="height: 0.68333em; vertical-align: 0em;"></span><span class="base textstyle uncramped"><span class="mord text textstyle uncramped"><span class="mord mathrm">int.MAX</span></span></span></span></span></span>, 
we will hit integer overflow problem.</p>
Following the thoughts based on the second idea, to avoid the overflow issue of the reverted number, what if we only 
revert half of the <span class="maths katex-rendered">i</mi>n</mi>t</mi></mtext></mrow><annotation encoding="application/x-tex">text{int}</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height: 0.66786em;"></span><span class="strut bottom" style="height: 0.66786em; vertical-align: 0em;"></span><span class="base textstyle uncramped"><span class="mord text textstyle uncramped"><span class="mord mathrm">int</span></span></span></span></span></span> number? After all, the reverse of the last half of the palindrome should be the same as the 
first half of the number, if the number is a palindrome.</p>
For example, if the input is 1221</code>, if we can revert the last part of the number "1221</strong>" from "21</strong>" to "12</strong>",
and compare it with the first half of the number "12", since 12 is the same as 12, we know that the number is a palindrome. </p>
Let's see how we could translate this idea into an algorithm.</p>
Algorithm</strong></p>
First of all we should take care of some edge cases. All negative numbers are not palindrome, for example: -123 is 
not a palindrome since the '-' does not equal to '3'. So we can return false for all negative numbers.</p>
Now let's think about how to revert the last half of the number. For number 1221</code>, if we do 1221 % 10</code>, we get the 
last digit 1</code>, to get the second to the last digit, we need to remove the last digit from 1221</code>, we could do so by 
dividing it by 10, 1221 / 10 = 122</code>. Then we can get the last digit again by doing a modulus by 10, 122 % 10 = 2</code>, and if we multiply the last digit by 10 and add the second last digit, 1 * 10 + 2 = 12</code>, it gives us the reverted number we want. Continuing this process would give us the reverted number with more digits. </p>
Now the question is, how do we know that we've reached the half of the number?</p>
Since we divided the number by 10, and multiplied the reversed number by 10, when the original number is less than the 
reversed number, it means we've processed half of the number digits.</p>
C#</strong></p>
</span>public</span> class</span> Solution</span> {</span>
    public</span> bool</span> IsPalindrome</span>(</span>int</span> x</span>)</span> {</span>
        // Special cases:</span>
        // As discussed above, when x &lt; 0, x is not a palindrome.</span>
        // Also if the last digit of the number is 0, in order to be a palindrome, </span>
        // the first digit of the number also needs to be 0.</span>
        // Only 0 satisfy this property.</span>
        if</span>(</span>x</span> &lt;</span> 0</span> ||</span> (</span>x</span> %</span> 10</span> ==</span> 0</span> &amp;&amp;</span> x</span> !=</span> 0</span>))</span> {</span>
            return</span> false</span>;</span>
        }</span>

        int</span> revertedNumber</span> =</span> 0</span>;</span>
        while</span>(</span>x</span> &gt;</span> revertedNumber</span>)</span> {</span>
            revertedNumber</span> =</span> revertedNumber</span> *</span> 10</span> +</span> x</span> %</span> 10</span>;</span>
            x</span> /=</span> 10</span>;</span>
        }</span>

        // When the length is an odd number, we can get rid of the middle digit by revertedNumber/10</span>
        // For example when the input is 12321, at the end of the while loop we get x = 12, revertedNumber = 123, </span>
        // since the middle digit doesn't matter in palidrome(it will always equal to itself), we can simply get rid of it.</span>
        return</span> x</span> ==</span> revertedNumber</span> ||</span> x</span> ==</span> revertedNumber</span>/</span>10</span>;</span>
    }</span>
}</span>
