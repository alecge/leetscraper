The data structure Trie (Prefix tree) and most common operations with it.</p>

Applications</h4>
Trie (we pronounce "try") or prefix tree is a tree data structure, which is used for retrieval of a key in a dataset of strings.
There are various applications of this very efficient data structure such as :</p>

There are several other data structures, like balanced trees and hash tables, which give us the possibility to search for a word in a dataset of strings. Then why do we need trie?
Although hash table has <span class="maths katex-rendered">O</mi>(</mo>1</mn>)</mo></mrow><annotation encoding="application/x-tex">O(1)</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height: 0.75em;"></span><span class="strut bottom" style="height: 1em; vertical-align: -0.25em;"></span><span class="base textstyle uncramped"><span class="mord mathit" style="margin-right: 0.02778em;">O</span>(</span><span class="mord mathrm">1</span>)</span></span></span></span></span> time complexity for looking for a key, it is not efficient in the following operations :</p>

Finding all keys with a common prefix.</li>
Enumerating a dataset of strings in lexicographical order.</li>
</ul>
Another reason why trie outperforms hash table, is that as hash table increases in size, there are lots of hash collisions and the search time complexity could deteriorate to <span class="maths katex-rendered">O</mi>(</mo>n</mi>)</mo></mrow><annotation encoding="application/x-tex">O(n)</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height: 0.75em;"></span><span class="strut bottom" style="height: 1em; vertical-align: -0.25em;"></span><span class="base textstyle uncramped"><span class="mord mathit" style="margin-right: 0.02778em;">O</span>(</span><span class="mord mathit">n</span>)</span></span></span></span></span>, where <span class="maths katex-rendered">n</mi></mrow><annotation encoding="application/x-tex">n</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height: 0.43056em;"></span><span class="strut bottom" style="height: 0.43056em; vertical-align: 0em;"></span><span class="base textstyle uncramped"><span class="mord mathit">n</span></span></span></span></span> is the number of keys inserted.
Trie could use less space compared to Hash Table when storing many keys with the same prefix.
In this case using trie has only <span class="maths katex-rendered">O</mi>(</mo>m</mi>)</mo></mrow><annotation encoding="application/x-tex">O(m)</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height: 0.75em;"></span><span class="strut bottom" style="height: 1em; vertical-align: -0.25em;"></span><span class="base textstyle uncramped"><span class="mord mathit" style="margin-right: 0.02778em;">O</span>(</span><span class="mord mathit">m</span>)</span></span></span></span></span> time complexity, where <span class="maths katex-rendered">m</mi></mrow><annotation encoding="application/x-tex">m</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height: 0.43056em;"></span><span class="strut bottom" style="height: 0.43056em; vertical-align: 0em;"></span><span class="base textstyle uncramped"><span class="mord mathit">m</span></span></span></span></span> is the key length.
Searching for a key in a balanced tree costs  <span class="maths katex-rendered">O</mi>(</mo>m</mi>log</mi>n</mi>)</mo></mrow><annotation encoding="application/x-tex">O(m log n)</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height: 0.75em;"></span><span class="strut bottom" style="height: 1em; vertical-align: -0.25em;"></span><span class="base textstyle uncramped"><span class="mord mathit" style="margin-right: 0.02778em;">O</span>(</span><span class="mord mathit">m</span>lo<span style="margin-right: 0.01389em;">g</span></span><span class="mord mathit">n</span>)</span></span></span></span></span> time complexity.</p>
<h4 id="trie-node-structure">Trie node structure</h4>
Trie is a rooted tree. Its nodes have the following fields:</p>

Maximum of <span class="maths katex-rendered">R</mi></mrow><annotation encoding="application/x-tex">R</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height: 0.68333em;"></span><span class="strut bottom" style="height: 0.68333em; vertical-align: 0em;"></span><span class="base textstyle uncramped"><span class="mord mathit" style="margin-right: 0.00773em;">R</span></span></span></span></span> links to its children, where each link corresponds to one of <span class="maths katex-rendered">R</mi></mrow><annotation encoding="application/x-tex">R</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height: 0.68333em;"></span><span class="strut bottom" style="height: 0.68333em; vertical-align: 0em;"></span><span class="base textstyle uncramped"><span class="mord mathit" style="margin-right: 0.00773em;">R</span></span></span></span></span> character values from dataset alphabet.
In this article we assume that <span class="maths katex-rendered">R</mi></mrow><annotation encoding="application/x-tex">R</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height: 0.68333em;"></span><span class="strut bottom" style="height: 0.68333em; vertical-align: 0em;"></span><span class="base textstyle uncramped"><span class="mord mathit" style="margin-right: 0.00773em;">R</span></span></span></span></span> is 26, the number of lowercase latin letters.</li>
Boolean field which specifies whether the node corresponds to the end of the key, or is just a key prefix.</li>
</ul>
<img alt="Representation of a key in trie" src="https://leetcode.com/media/original_images/208_Node.png" width="539px"></p>
Figure 6. Representation of a key "leet" in trie.</em></p>
Java</strong></p>
</span>class</span> TrieNode</span> {</span>

    // R links to node children</span>
    private</span> TrieNode</span>[]</span> links</span>;</span>

    private</span> final</span> int</span> R</span> =</span> 26</span>;</span>

    private</span> boolean</span> isEnd</span>;</span>

    public</span> TrieNode</span>()</span> {</span>
        links</span> =</span> new</span> TrieNode</span>[</span>R</span>];</span>
    }</span>

    public</span> boolean</span> containsKey</span>(</span>char</span> ch</span>)</span> {</span>
        return</span> links</span>[</span>ch</span> -</span>'a'</span>]</span> !=</span> null</span>;</span>
    }</span>
    public</span> TrieNode</span> get</span>(</span>char</span> ch</span>)</span> {</span>
        return</span> links</span>[</span>ch</span> -</span>'a'</span>];</span>
    }</span>
    public</span> void</span> put</span>(</span>char</span> ch</span>,</span> TrieNode</span> node</span>)</span> {</span>
        links</span>[</span>ch</span> -</span>'a'</span>]</span> =</span> node</span>;</span>
    }</span>
    public</span> void</span> setEnd</span>()</span> {</span>
        isEnd</span> =</span> true</span>;</span>
    }</span>
    public</span> boolean</span> isEnd</span>()</span> {</span>
        return</span> isEnd</span>;</span>
    }</span>
}</span>

Two of the most common operations in a trie are insertion of a key and search for a key.</p>
<h4 id="insertion-of-a-key-to-a-trie">Insertion of a key to a trie</h4>
We insert a key by searching into the trie. We start from the root and search a link, which corresponds to the first key character. There are two cases :</p>

A link exists. Then we move down the tree following the link to the next child level. The algorithm continues with searching for the next key character.</li>
A link does not exist. Then we create a new node and link it with the parent's link matching the current key character.
We repeat this step until we encounter the last character of the key, then we mark the current node as an end node and the algorithm finishes.</li>
</ul>
<img alt="Insertion of keys into a trie" src="https://leetcode.com/media/original_images/208_TrieInsert.png" width="539px"></p>
Figure 7. Insertion of keys into a trie.</em></p>
Java</strong></p>
</span>class</span> Trie</span> {</span>
    private</span> TrieNode</span> root</span>;</span>

    public</span> Trie</span>()</span> {</span>
        root</span> =</span> new</span> TrieNode</span>();</span>
    }</span>

    // Inserts a word into the trie.</span>
    public</span> void</span> insert</span>(</span>String</span> word</span>)</span> {</span>
        TrieNode</span> node</span> =</span> root</span>;</span>
        for</span> (</span>int</span> i</span> =</span> 0</span>;</span> i</span> &lt;</span> word</span>.</span>length</span>();</span> i</span>++)</span> {</span>
            char</span> currentChar</span> =</span> word</span>.</span>charAt</span>(</span>i</span>);</span>
            if</span> (!</span>node</span>.</span>containsKey</span>(</span>currentChar</span>))</span> {</span>
                node</span>.</span>put</span>(</span>currentChar</span>,</span> new</span> TrieNode</span>());</span>
            }</span>
            node</span> =</span> node</span>.</span>get</span>(</span>currentChar</span>);</span>
        }</span>
        node</span>.</span>setEnd</span>();</span>
    }</span>
}</span>

</ul>
In the worst case newly inserted key doesn't share a prefix with the the keys already inserted in the trie. new nodes, which takes us  <span class="maths katex-rendered">O</mi>(</mo>m</mi>)</mo></mrow><annotation encoding="application/x-tex">O(m)</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height: 0.75em;"></span><span class="strut bottom" style="height: 1em; vertical-align: -0.25em;"></span><span class="base textstyle uncramped"><span class="mord mathit" style="margin-right: 0.02778em;">O</span>(</span><span class="mord mathit">m</span>)</span></span></span></span></span> space.</p>
<h4 id="search-for-a-key-in-a-trie">Search for a key in a trie</h4>
Each key is represented in the trie as a path from the root to the internal node or leaf.
We start from the root with the first key character. We examine the current node for a link corresponding to the key character. There are two cases :</p>

A link exist. We move to the next node in the path following this link, and proceed searching for the next key character.</li>

A link does not exist. If there are no available key characters and current node is marked as isEnd</code> we return true. Otherwise there are possible two cases in each of them we return false :</p>

There are key characters left, but it is impossible to follow the key path in the trie, and the key is missing.</li>
No key characters left, but current node is not marked as isEnd</code>. Therefore the search key is only a prefix of another key in the trie.</li>
</ul>
</li>
</ul>
<img alt="Search of a key in a trie" src="https://leetcode.com/media/original_images/208_TrieSearchKey.png" width="539px"></p>
Figure 8. Search for a key in a trie.</em></p>
Java</strong></p>
</span>class</span> Trie</span> {</span>
    ...</span>

    // search a prefix or whole key in trie and</span>
    // returns the node where search ends</span>
    private</span> TrieNode</span> searchPrefix</span>(</span>String</span> word</span>)</span> {</span>
        TrieNode</span> node</span> =</span> root</span>;</span>
        for</span> (</span>int</span> i</span> =</span> 0</span>;</span> i</span> &lt;</span> word</span>.</span>length</span>();</span> i</span>++)</span> {</span>
           char</span> curLetter</span> =</span> word</span>.</span>charAt</span>(</span>i</span>);</span>
           if</span> (</span>node</span>.</span>containsKey</span>(</span>curLetter</span>))</span> {</span>
               node</span> =</span> node</span>.</span>get</span>(</span>curLetter</span>);</span>
           }</span> else</span> {</span>
               return</span> null</span>;</span>
           }</span>
        }</span>
        return</span> node</span>;</span>
    }</span>

    // Returns if the word is in the trie.</span>
    public</span> boolean</span> search</span>(</span>String</span> word</span>)</span> {</span>
       TrieNode</span> node</span> =</span> searchPrefix</span>(</span>word</span>);</span>
       return</span> node</span> !=</span> null</span> &amp;&amp;</span> node</span>.</span>isEnd</span>();</span>
    }</span>
}</span>

<h4 id="search-for-a-key-prefix-in-a-trie">Search for a key prefix in a trie</h4>
The approach is very similar to the one we used for searching a key in a trie. We traverse the trie from the root, till there are no characters left in key prefix or it is impossible to continue the path in the trie with the current key character. The only difference with the mentioned above search for a key</code> algorithm is that when we come to an end of the key prefix, we always return true. We don't need to consider the isEnd</code> mark of the current trie node, because we are searching for a prefix of a key, not for a whole key.</p>

Figure 9. Search for a key prefix in a trie.</em></p>
Java</strong></p>
</span>class</span> Trie</span> {</span>

    // that starts with the given prefix.</span>
    public</span> boolean</span> startsWith</span>(</span>String</span> prefix</span>)</span> {</span>
        TrieNode</span> node</span> =</span> searchPrefix</span>(</span>prefix</span>);</span>
        return</span> node</span> !=</span> null</span>;</span>
