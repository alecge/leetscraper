<h4 id="approach-using-sub-query-and-not-in-clause-accepted">Approach: Using sub-query and NOT IN</code> clause [Accepted]</h4>
Algorithm</strong></p>
If we have a list of customers who have ever ordered, it will be easy to know who never ordered.</p>
We can use the following code to get such list.</p>
</span>select</span> customerid</span> from</span> orders</span>;</span>
</pre></div>


Then, we can use NOT IN</code> to query the customers who are not in this list.</p>
MySQL</strong></p>
</span>select</span> customers</span>.</span>name</span> as</span> 'Customers'</span>
from</span> customers</span>
where</span> customers</span>.</span>id</span> not</span> in</span>
(</span>
    select</span> customerid</span> from</span> orders</span>
);</span>
</pre></div>
