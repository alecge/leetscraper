<h4 id="approach-1-brute-force-time-limit-exceeded">Approach #1 Brute Force [Time Limit Exceeded]</h4>
Algorithm</strong></p>
The simplest approach is to try to find all increasing subsequences and then returning the maximum length of longest increasing subsequence. In order to
do this, we make use of a recursive function <span class="maths katex-rendered">l</mi>e</mi>n</mi>g</mi>t</mi>h</mi>o</mi>f</mi>L</mi>I</mi>S</mi></mrow><annotation encoding="application/x-tex">lengthofLIS</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height: 0.69444em;"></span><span class="strut bottom" style="height: 0.88888em; vertical-align: -0.19444em;"></span><span class="base textstyle uncramped"><span class="mord mathit" style="margin-right: 0.01968em;">l</span><span class="mord mathit">e</span><span class="mord mathit">n</span><span class="mord mathit" style="margin-right: 0.03588em;">g</span><span class="mord mathit">t</span><span class="mord mathit">h</span><span class="mord mathit">o</span><span class="mord mathit" style="margin-right: 0.10764em;">f</span><span class="mord mathit">L</span><span class="mord mathit" style="margin-right: 0.07847em;">I</span><span class="mord mathit" style="margin-right: 0.05764em;">S</span></span></span></span></span> which returns the length of the LIS possible from the current element(corresponding to <span class="maths katex-rendered">c</mi>u</mi>r</mi>p</mi>o</mi>s</mi></mrow><annotation encoding="application/x-tex">curpos</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="strut" style="height: 0.43056em;"></span><span class="strut bottom" style="height: 0.625em; vertical-align: -0.19444em;"></span><span class="base textstyle uncramped"><span class="mord mathit">c</span><span class="mord mathit">u</span><span class="mord mathit" style="margin-right: 0.02778em;">r</span><span class="mord mathit">p</span><span class="mord mathit">o</span><span class="mord mathit">s</span></span></span></span></span>)
 onwards(including the current element). Inside each function call, we consider two cases:</p>


The current element is larger than the previous element included in the LIS. In this case, we can include the current element in the LIS. Thus, we find out the
length of the LIS obtained by including it. Further, we also find out the length of LIS possible by not including the current element in the LIS. The value returned
by the current function call is, thus, the maximum out of the two lengths.</p>
</li>

The current element is smaller than the previous element included in the LIS. In this case, we can't include the current element in the LIS. Thus, we find out only
the length of the LIS possible by not including the current element in the LIS, which is returned by the current function call.</p>
</li>
</ol>

</span>public</span> class</span> Solution</span> {</span>

    public</span> int</span> lengthOfLIS</span>(</span>int</span>[]</span> nums</span>)</span> {</span>
        return</span> lengthofLIS</span>(</span>nums</span>,</span> Integer</span>.</span>MIN_VALUE</span>,</span> 0</span>);</span>
    }</span>

    public</span> int</span> lengthofLIS</span>(</span>int</span>[]</span> nums</span>,</span> int</span> prev</span>,</span> int</span> curpos</span>)</span> {</span>
        if</span> (</span>curpos</span> ==</span> nums</span>.</span>length</span>)</span> {</span>
            return</span> 0</span>;</span>
        }</span>
        int</span> taken</span> =</span> 0</span>;</span>
        if</span> (</span>nums</span>[</span>curpos</span>]</span> &gt;</span> prev</span>)</span> {</span>
            taken</span> =</span> 1</span> +</span> lengthofLIS</span>(</span>nums</span>,</span> nums</span>[</span>curpos</span>],</span> curpos</span> +</span> 1</span>);</span>
        }</span>
        int</span> nottaken</span> =</span> lengthofLIS</span>(</span>nums</span>,</span> prev</span>,</span> curpos</span> +</span> 1</span>);</span>
        return</span> Math</span>.</span>max</span>(</span>taken</span>,</span> nottaken</span>);</span>
    }</span>
}</span>

<h4 id="approach-2-recursion-with-memorization-memory-limit-exceeded">Approach #2 Recursion with memorization [Memory Limit Exceeded]</h4>

</span>public</span> class</span> Solution</span> {</span>
    public</span> int</span> lengthOfLIS</span>(</span>int</span>[]</span> nums</span>)</span> {</span>
        int</span>[]</span> dp</span> =</span> new</span> int</span>[</span>nums</span>.</span>length</span>];</span>
        int</span> len</span> =</span> 0</span>;</span>
        for</span> (</span>int</span> num</span> :</span> nums</span>)</span> {</span>
            int</span> i</span> =</span> Arrays</span>.</span>binarySearch</span>(</span>dp</span>,</span> 0</span>,</span> len</span>,</span> num</span>);</span>
            if</span> (</span>i</span> &lt;</span> 0</span>)</span> {</span>
                i</span> =</span> -(</span>i</span> +</span> 1</span>);</span>
            }</span>
            dp</span>[</span>i</span>]</span> =</span> num</span>;</span>
            if</span> (</span>i</span> ==</span> len</span>)</span> {</span>
                len</span>++;</span>
            }</span>
        }</span>
        return</span> len</span>;</span>
    }</span>
}</span>

Note: Arrays.binarySearch() method returns index of the search key, if it is contained in the array, else it returns (-(insertion point) - 1).
